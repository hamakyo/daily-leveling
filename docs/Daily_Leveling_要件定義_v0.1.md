# Daily Leveling 要件定義 v0.1

## 1. プロダクト概要

### 1-1. プロジェクト名
Daily Leveling

### 1-2. 概要
Daily Leveling は、日々の習慣を記録し、達成率・継続状況・傾向を可視化することで、行動改善を継続しやすくする Web アプリである。  
単なる ToDo 管理ではなく、**「毎日少しずつレベルを上げていく感覚」** を提供することを主目的とする。

### 1-3. コンセプト
- 習慣の継続をゲームのレベル上げ感覚で支援する
- 月間グリッドで生活の進捗を一覧できる
- 達成率・継続日数・傾向を可視化する
- 厳しい評価よりも、蓄積と再開しやすさを重視する

### 1-4. ターゲットユーザー
- 自己改善や習慣化に関心がある個人ユーザー
- スプレッドシートや Notion で習慣管理しているが、入力や可視化に不満がある人
- スマホでも PC でも気軽に日次記録したい人

---

## 2. MVP の目的

### 2-1. MVP で検証したいこと
- 月間グリッド中心の UI が、日々の記録を継続しやすいか
- 習慣の可視化により、ユーザーが達成感と継続意欲を得られるか
- PC / モバイルの両方でストレスなく入力できるか

### 2-2. MVP の成功基準
- ユーザーが習慣を登録し、日次で記録できる
- 月単位で達成状況を振り返れる
- 最低限のゲーム化要素により、継続したくなる体験を提供できる

---

## 3. スコープ

### 3-1. MVP に含めるもの
- Google SSO によるログイン
- 習慣の作成・編集・削除
- 習慣の並び替え
- 習慣のアクティブ / 非アクティブ切り替え
- 日次チェックイン（達成 / 未達）
- 月間グリッド表示
- 今日ビュー表示
- 月間達成率の表示
- 習慣別達成率の表示
- 連続達成日数の表示
- 初回オンボーディング
- テンプレートからの初期習慣登録

### 3-2. MVP に含めないもの
- 単発タスク管理
- 回数 / 時間 / 数値入力による記録
- 通知・リマインド
- XP / レベル / バッジ / 通貨
- ソーシャル共有
- AI による週次レビュー
- 外部カレンダー連携
- PWA / ウィジェット
- 共同利用機能

## 4. プロダクト方針

### 4-1. 習慣専用アプリとして扱う
MVP では「習慣」のみを扱い、単発タスクや締切付きタスクはスコープ外とする。

### 4-2. 記録単位
- 1 習慣につき 1 日 1 回の記録
- 記録値は boolean（達成 / 未達）
- 達成条件の詳細はユーザーが自己定義する

### 4-3. 頻度ルール
MVP では以下に対応する。
- 毎日
- 曜日指定

以下は将来対応とする。
- 週 n 回
- 月 n 回
- 条件付きスケジュール

### 4-4. ゲーム化方針
MVP では軽量なゲーム化のみ採用する。
- 月間達成率
- 習慣別達成率
- 連続達成日数
- 完了数の蓄積

未達成時の懲罰的表現は避け、再開しやすい設計にする。

## 5. 主要ユースケース

### 5-1. ログイン
1. ユーザーが Google SSO でログインする
2. 初回ログイン時はオンボーディングへ遷移する
3. 2 回目以降はダッシュボードへ遷移する

### 5-2. 習慣登録
1. ユーザーが習慣名・絵文字・色・頻度を入力する
2. 習慣を保存する
3. ダッシュボードに反映される

### 5-3. 日次チェック
1. ユーザーが今日ビューまたは月間グリッドを開く
2. 対象習慣のセルをタップ / クリックする
3. 達成 / 未達が切り替わる
4. 集計が更新される

### 5-4. 月間振り返り
1. ユーザーが月を切り替える
2. 月間グリッド・達成率・習慣別進捗を確認する
3. 継続できている習慣と崩れている習慣を把握する

### 5-5. 習慣編集
1. ユーザーが習慣設定画面を開く
2. 習慣名・色・絵文字・頻度・並び順などを変更する
3. 変更内容が即時または保存後に反映される

## 7. 機能要件

### 7-1. 認証
- Google SSO に対応する
- 未認証ユーザーはアプリ本体へアクセスできない
- 初回ログイン判定を保持する

### 7-2. 習慣管理
- 習慣を作成できる
- 習慣を編集できる
- 習慣を削除できる
- 習慣をアクティブ / 非アクティブにできる
- 習慣の表示順を変更できる
- 習慣に色・絵文字を設定できる
- 頻度として毎日または曜日指定を設定できる

### 7-3. 日次ログ
- 日付ごとに達成 / 未達を記録できる
- 同一ユーザー・同一習慣・同一日付の記録は一意である
- 未来日は記録不可
- 過去日の修正は可能
- 頻度対象外の日は未入力扱いとし、達成率計算対象から除外する

### 7-4. ダッシュボード表示
- 月単位でデータを表示できる
- 日別達成率を表示できる
- 習慣別達成率を表示できる
- 月間総達成率を表示できる
- 現在の連続達成日数を表示できる

### 7-5. オンボーディング
- 初回ログイン時に表示する
- テンプレートから初期習慣を投入できる
- 手動で習慣を追加できる

## 8. 業務ルール / データルール

### 8-1. 日付ルール
- 1 日の区切りはユーザーのローカル日付基準とする
- DB には UTC ベースで保存し、表示時にユーザーのタイムゾーンへ変換する
- ユーザーごとにタイムゾーンを保持する

### 8-2. 習慣ログの一意性
- `user_id + habit_id + date` は一意であること

### 8-3. 頻度対象日のみ評価
- 毎日設定の習慣は全日が対象
- 曜日指定の習慣は指定曜日のみ対象
- 対象外の日は達成率の母数に含めない

### 8-4. 未来日の扱い
- 未来日は記録不可
- 表示は可能だが入力操作は無効化する

### 8-5. 削除の扱い
- MVP では論理削除または `is_active = false` による非表示を基本方針とする
- 完全削除の挙動は実装時に明確化する

---

## 9. 非機能要件

### 9-1. パフォーマンス
- 月間グリッドは体感 1 秒前後で表示されること
- 達成トグル操作は体感即時で反映されること
- モバイル環境でも操作に支障がないこと

### 9-2. 可用性
- 通常利用時間帯に安定してアクセス可能であること
- 書き込み失敗時は再試行またはエラー表示を行うこと

### 9-3. セキュリティ
- ユーザーごとのデータは完全に分離されること
- 認証必須であること
- API / BFF 層で他人のデータへアクセスできないこと
- 通信は HTTPS 前提とする

### 9-4. UX
- ワンタップ / ワンクリックで記録できること
- 画面の主要情報がひと目で把握できること
- 未達成時も再開しやすい心理的設計にすること

## 11. 集計指標

### 11-1. 月間総達成率
対象月における、対象習慣の対象日総数に対する達成数の割合

### 11-2. 習慣別達成率
対象月における、各習慣ごとの達成率

### 11-3. 日別達成率
各日付における、対象習慣数に対する達成数の割合

### 11-4. 連続達成日数
連続して達成が続いた日数  
※ MVP では計算定義をシンプルにし、詳細ルールは設計フェーズで明確化する

## 12. 技術方針

### 12-1. 想定構成
- フロントエンド / アプリ実行基盤: Cloudflare Workers
- データベース: PostgreSQL
- DB 接続高速化: Hyperdrive
- 認証: Google SSO
- キャッシュや通知基盤: MVP では未採用

### 12-2. 技術方針メモ
- 本体 DB は Postgres を採用し、リレーショナルな集計を素直に扱う
- 集計は MVP 段階では都度クエリで計算する
- 事前集計テーブルやキャッシュは必要になってから導入する

## 16. 要約

Daily Leveling の MVP は、**習慣記録に特化した軽量な習慣トラッカー** として定義する。  
中核価値は以下の 3 点である。

- 月間グリッドによる一覧性
- 日次記録のしやすさ
- 達成率・継続状況の可視化による継続支援

MVP では機能を広げすぎず、**習慣記録の体験そのものを成立させること** を最優先とする。
